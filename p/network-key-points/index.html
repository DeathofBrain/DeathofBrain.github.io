计算机网络体系结构

本章内容重点：三种交换（计算）、OSI分层、TCP/IP分层（记忆）、各种指标与单位（计算）

▲概述

▲概念

定义：将地理位置不同，具有独立功能的多台计算机以及外部设备通过通信线路连接起来，实现资源共享和信息传递的操作系统。

组成

需要知道哪些东西是哪个部分的就可以了

  • 从组成部分看，计算机网络主要由硬件、软件、协议三大部分组成。
    • 硬件：由主机、通信链路、交换设备和网卡等组成
    • 软件：包括email程序、ftp程序等
    • 协议：计算机网络的核心，规定了网络传输数据时所遵循的规范。
  • 从工作方式看，可分为边缘部分和核心部分
    • 边缘部分：由所有连接到互联网上的供用户直接使用的主机构成，进行通信和资源共享。
    • 核心部分：大量网络和连接着的路由器组成。
  • 从功能组成看，可分为通信子网和资源子网。
    • 通信子网：各种传输介质、通信设备和相应网络协议组成。
      • 使网络具有数据传输、交换、控制和存储的能力，实现数据通信。
    • 资源子网：实现资源共享的设备及其软件的集合
      • 向网络用户提供共享其他计算机上的硬件资源、软件资源、数据资源的服务。

功能

  1. 数据通信：最基本和最重要的功能，实现联网计算机之间信息的传输。
  2. 资源共享：共享软件、硬件、数据，让网络中资源互通有无。
  3. 分布式处理：将某个复杂的任务分配给网络中其它计算机系统，提高整个系统的利用率。
  4. 提高可靠性：网络中的各台计算机可以互为替代机。
  5. 负载均衡：将工作任务均衡地分配给网络中各台计算机。

🌟电路交换、报文交换和分组交换

计网计算题重点，可结合到多种题型

路由器这个网络层设备在网络核心部分起到重要作用。它对收到的分组进行存储转发实现分组交换。分组交换是现代网络最常用的交换方式。

从通信资源分配的角度看，交换就是按照某种方式动态地分配传输线路的资源。

分组交换由电路交换与报文交换发展而来。

  • 电路交换（典型：电话网）

    • 特点：在进行数据传输前，两个用户之间必须先建立一条专用的物理通信路径，且传输过程中，这一路径始终被两个用户独占，直到通信结束被释放。

    • 分为三步：建立连接（开始占用通信资源）、传输数据、释放连接。

    • 示意图：

    • 整条电路上任何节点都不进行存储，直接转发，不存在存储转发所耗费的时间。

      这一点需要记住。

    • 优点：通信时延小、有序传输、没有冲突、实时性强。

    • 缺点：建立连接时间长、线路利用率低、灵活性差、难以实现差错控制。

      灵活性差：物理通路中，任何一点出现故障，就必须重新建立连接。

      难以实现差错控制：中间节点不具备存储和检验数据的能力，无法发现并纠正错误。

事实上，计算机之间的数据传输往往是突发式的（高频、少量数据），若使用电路交换，那么整个通信链路资源的利用率往往不到10%，甚至只有1%。

  • 报文交换——分组交换的前身

    • 数据交换的单位是报文。报文由用户数据、源地址和目的地址等信息组成。

    • 采用存储转发技术，整个报文先传送到相邻的节点，全部存储后查找转发表，转发到下一个节点，如此反复，直到报文到达目的节点。

      全部存储——报文交换被淘汰的原因

    • 不同于电路交换，每个报文都可单独选择到达目的端的路径。示例图如下：

    • 优点：无建立连接时延、灵活分配线路、线路利用率高、支持差错控制。

      这些也是存储转发本身的优点。

    • 缺点：转发时延高、缓存开销大、错误处理低效。

      交换节点要把报文都存储完才能查找转发表转发到下一节点，若是报文太大，缓存开销也大，并且一旦丢包，重传代价也大。

分组交换，就是在报文交换存储转发的基础上，将报文划分成若干较小等长数据段，每个数据段前面添加一些由控制信息组成的首部，构成分组（Packet）。

  • 分组交换

    • 流程

      源主机发送分组，分组交换网中的交换机收到分组后，先缓存，然后从首部提取目的地址，借由转发表，发送给下一个交换机，周而复始，直到到达目的主机。

      本质上和报文交换一样

    • 优点：存储转发开销小、传输效率高、减少出错概率与代价。

      分组是逐个传输的，可以使后一个分组的存储操作与前一个分组的转发操作并行，形成流水线，后面的图会详细表达。

    • 缺点：存在存储转发时延、需要传输额外信息、可能出现失序、丢失或重复分组的情况。

      失序丢包等问题是借由上层，或使用虚电路服务解决的，先不必深究。

  • 三种交换方式的比较

    • 当要连续传输大量的数据，并且传送时间远大于连接建立时间时，采用电路交换更合适。

      然而，现时代，数据的传送时间肯定小于连接建立时间。

    • 从提高整个网络的信道利用率看，报文交换和分组交换更合适

      • 其中，分组交换比报文交换时延更小，更加灵活，尤其适合突发式数据传送。

    • 三种传输时序图

      为什么是斜的：电磁波在信道中传播需要花费时间（即使非常短），这部分时间也叫传播时延。有些题明确指出传播时延忽略不计时，那就必须要画平了。

分类

记一下拓扑结构就好了

  • 分布范围：广域网、城域网、局域网、个人区域网

  • 传输技术：广播式网络、点对点网络

  • 拓扑规则：

    • 总线型：优点是建网容易、增删节点方便、节省线路；缺点是重负载时通信效率不高，总线任意一处对故障敏感。
    • 星形网络：优点是便于集中控制和管理；缺点是成本高、中央设备对故障敏感。
    • 环形网络：典型例子是令牌环局域网。
    • 网状网络：每个节点至少有两条路径与其他节点相连，常用于广域网。优点是可靠性高，缺点是控制复杂、线路成本高。

  • 使用者：公用网、专用网

  • 传输介质：有线网络、无线网络。

🌟性能指标

这个不会就真埋了吧

  • 速率：节点在数字信道上传送数据的速率，也称数据传输速率、数据率或比特率。

    • 单位：b/s或bit/s
    • 数据率较高时，可用Kb/s、Mb/s、Gb/s等。

    1. 放到常见情形，可以对应下载速率或上传速率

    2. 在计网中，K=10^3，M=10^6，G=10^9，幂数每一级+3

  • 带宽：在计网中，表示网络等通信线路所能传送数据的能力。

    也就是最高数据传输速率

    • 单位：bit/s

  • 吞吐量：单位时间内通过某个网络（或信道、接口）的实际数据量。

    • 常用于对实际网络的测量，以便获知有多少数据量能够通过网络。
    • 一条由多端链路组成的信道，其带宽取决于带宽最小的那段链路。

  • 🌟时延：指数据从网络的一段传送到另一段所需的总时间。由四部分组成：发送时延、传播时延、处理时延和排队时延。

    • 发送时延：又称传输时延，节点将分组的所有比特推向链路所花的时间。

      即从发送分组的第一个比特算起，到该分组的最后一个比特发送完毕所花的时间。

      • 公式：发送时延 = 分组长度 / 发送速率
    • 传播时延：电磁波在信道中传播一定距离所花时间，即一个比特从链路的一端到另一段所需时间。
      • 公式：传播时延 = 信道长度 / 电磁波在信道上的传播速率

    传输时延和传播时延一定要区分开，不然没分了

    • 处理时延：分组在交换节点为存储转发而进行的一些必要处理所花的时间。
    • 排队时延：分组在路由器的输入队列或输出队列中排队等待所花时间。

    处理时延与排队时延一般忽略不计，或者题目直接给出时间。 因此，总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延

  • 时延带宽积：指发送端发送的第一个比特即将到达终点时，发送端已经发送了多少比特。

    • 时延带宽积 = 传播时延 x 信道带宽

    • 想象成圆柱形管道，长度为传播时延，横截面积表示链路带宽，则时延带宽积表示该管道可以容纳的比特数量。

  • 🌟往返时延（RTT）：指从发送端发出一个短分组，到发送端收到来自接收端的确认（接收端收到数据后立刻发送确认）总共经历的时间。

    RTT包括各中间节点的处理时延、排队时延以及转发数据时的发送时延。

  • 信道利用率：指出信道有数据通过时间占总时间的百分比

    • 信道利用率 = 有数据通过时间 / （有数据通过时间 + 无数据通过时间）
    • 信道利用率并非越高越好，太低会浪费资源，太高会造成拥堵。

🌟体系结构、分层模型

概念

计算机网络的各层及其协议的集合称为网络的体系结构。

换言之，计算机网络的体系结构就是这个计算机网络及其所应完成的功能的精确定义。

要注意，这些功能如何实现，是体系结构的实现问题。体系结构是抽象的，它不在乎如何去实现，实现问题是具体的。

分层

计网体系结构通常都是可分层的，它将复杂的大系统分成若干较容易实现的层次。

分层的基本原则：

  • 每层都实现一种相对独立的功能
  • 各层之间接口清晰、易于理解、相互交流尽可能少
  • 各层功能的精确定义独立于具体实现
  • 保持下层对上层的独立性，上层单向使用下层提供的服务
  • 整个分层结构应能促进标准化

实体

实体，指任何可发送或接收信息的硬件或软件进程，通常是某个特定的软件模块。

在网络分层结构中，第n层的活动元素通常称为第n层实体。

不同终端的同一层称为对等层，同一层的实体称为对等实体。

🌟第n层向第n+1层提供服务，这些服务也包括第n层及以下各层提供的服务。

就这一句话值得去背

第n层的实体称为服务提供者，服务提供者上一层的实体称为服务用户。

数据单元

  • 协议数据单元（PDU）：对等层之间传送的数据单位，第n层的PDU称为n-PDU
    • 各层的PDU都分为服务数据单元和协议控制信息两个部分
  • 服务数据单元（SDU）：层与层之间交换的数据单位，第n层的SDU称为n-SDU
  • 协议控制信息（PCI）：控制协议操作的信息。第n层的PCI称为n-PCI

具体三者的关系如图。

层次结构的含义

  • 第n层实体不仅要使用下面一层（n-1层）的服务来实现自身定义的功能，还要向上面一层（n+1层）提供本层的服务，并且该服务是第n层及其下面各层提供的服务总和。
  • 最低层（物理层）只提供服务，是整个层次结构的基础
  • 最高层（应用层）向用户提供服务
  • 上一层只能通过相邻层间接口使用下一层的服务，而不能调用其他层的服务。

🌟协议、接口、服务的概念

🌟协议

这个不会可以埋了

网络协议：为了在网络中进行数据交换而建立的通信规则、标准或约定，是控制在对等实体之间进行通信规则的集合，是水平的。

自然，不对等实体之间是没有协议的。就像A的网络层和B的传输层之间就没有协议这一说法。

🌟协议三要素：语法、语义、时序

  • 语法：数据与控制信息的格式
  • 语义：定义需要发出什么控制信息、完成什么动作、做出什么回答
  • 时序：事件实现顺序的详细说明，即执行各种操作的条件、时序关系等

接口

同一节点内 相邻两层间 实体交换信息的逻辑接口称为服务访问点（SAP）。

当然，每一层只能在紧邻层之间定义接口，而不能跨层定义接口。

请注意：服务是通过SAP提供给上层使用的，也就是说，第n层的SAP就是第n+1层可以访问第n层服务的地方。

服务

服务是指下层为紧邻上层提供的功能调用，是垂直的。

对等实体在协议的控制下，使得本层能为上层提供服务，但要实现本层协议，还需要使用下层提供的服务。

注意：并非在一层内完成的全部功能都称为服务，只有那些能够被高一层实体调用的功能才称为服务。

协议与服务区别

  • 协议是水平的，服务是垂直的
  • 只有本层协议的实现才能保证向上一层提供服务
  • 下面的协议对上层的服务用户是透明的，仅提供服务。

协议、接口、服务三者之间的关系

计网提供的服务

  • 面向连接服务与无连接服务
    • 面向连接服务：通信双方必须建立连接（TCP由传输层负责），分配资源，传输结束后释放连接和占用资源。
      • 代表协议：TCP（传输层）
    • 无连接服务：通信双方不需要建立连接，需要发送数据可直接发送，将每个带有目的地址的包传送到线路上，由系统选定路线进行传输
      • 常被描述为尽最大努力交付，是不可靠的服务
      • 代表协议：IP（网络层）、UDP（传输层）
  • 可靠服务与不可靠服务
    • 可靠服务：网络具有纠错检错、应答机制，保证数据正确、可靠地传送到目的地。
    • 不可靠服务：网络尽量让数据正确、可靠地传送到目的地，尽力而为。
      • 可以由更高层来保障网络的正确与可靠

      IP就是不可靠的网络层协议，但是TCP作为传输层的协议，可以借助这个不可靠的服务，向上层提供可靠的服务。

  • 有应答服务和无应答服务
    • 有应答服务：接收方在收到数据时，向发送方发送应答
      • 应答可以是肯定应答，也可以是否定应答。接收的数据有错误时，即发送否定应答，以此让发送者重发
      • 这个应答由系统内部自动实现，而不需要用户实现
    • 无应答服务：接收方在收到数据时，不发送应答
      • 若需要应答，则需要高层实现

      WWW服务，客户端收到服务器发送的页面文件后，不给出回答

🌟ISO/OSI模型和TCP/IP模型

OSI模型

ISO提出的网络体系结构模型，称为OSI模型，有七层，自底向上依次为：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。

七层都有什么，要背下来，同样，接下来的四层TCP/IP模型也要背下来，当然还有特供五层折中模型更要背下来

接下来的各层，在后续章节基本都要讲述，这里只是简单介绍

  • 物理层

    物理层的传输单位是比特，功能是在物理介质上为数据端设备透明地传输原始比特流。

    • 通信链路与通信节点的连接需要一些电路接口，物理层规定了这些接口的一些参数，如机械形状尺寸、引脚数量和排列等。
    • 物理层规定了通信链路上所传输的信号的意义和电气特征。

    物理介质并不在物理层协议之中

  • 数据链路层

    • 数据链路层的传输单位是帧
    • 两台主机之间的数据传输总是在一段一段的链路上进行的，使用专门的链路层协议。

      加强物理层功能，将物理层提供的可能出错的物理连接改造成逻辑上无差错的数据链路。

    • 数据链路层将网络层交付的IP分组封装成帧，可靠地传输到相邻节点的网络层，实现节点间差错控制、流量控制的功能。

      差错控制：物理层传输时，可能会发生比特位翻转错误等。差错控制可以检测出这些差错（CRC算法等），将收到的错误信息丢弃。

      流量控制：假设A的网速是千兆，B的网速是百兆，若不加以控制，则B会丢失许多来不及接收的数据，造成传输线路效率下降。流量控制可以协调两个节点的速率，使A与B之间的发送速率对上。当然，最终线路肯定是取决于速率低的一方。

    • 广播式网络（例如Wi-Fi）在这一层还要处理诸如控制对共享信道的访问等新的问题。

  • 网络层

    • 网络层的传输单位是数据报。这层关心的是通信子网的运行控制，主要任务是将网络层的PDU（分组）从源主机到目的主机，为分组交换网上不同主机提供通信服务。
    • 关键问题：对分组进行路由选择。
    • 实现功能：流量控制、拥塞控制、差错控制、网际互联等。

      当有多条可选择的路由线路时，网络层会根据网络情况，利用相应路由算法计算出一条合适的路径，使这个分组可以顺利地到达目标节点，如图。

      拥塞控制：当上图的节点都来不及接收分组，会导致大量分组丢失，网络处于拥塞状态。此时网络层要采取措施来缓解这种拥塞，这就是拥塞控制。

    • 在OSI中，网络层提供有连接可靠的虚电路服务，也提供无连接不可靠的数据报服务。

      请注意，在TCP/IP模型中，网络层仅提供无连接不可靠的数据报服务。

  • 传输层

    • 负责相互通信的主机中两个进程之间的通信，提供的是端到端服务，如流量控制、差错控制、连接建立与释放、可靠传输管理等服务。
    • 在OSI模型中，传输层只提供有连接可靠的服务。

      同样，在TCP/IP模型中，传输层提供有连接可靠服务（TCP），也提供无连接不可靠服务（UDP）。

    • 点到点：主机和主机之间。
    • 端到端：运行在不同主机之间两个进程之间的通信，一个进程用一个端口来标识。
    • 通过传输层的屏蔽，高层用户看不到通信子网的交替和变化。因为一台主机可同时运行多个进程，故传送层具有复用和分用的功能。
      • 复用：多个应用层进程可同时运行多个进程
      • 分用：传送层将收到的信息分别交付给应用层中对应的进程。

  • 会话层

    • 作用：为表示层实体或用户进程建立连接，在连接上有序传输数据。
    • 负责管理主机间会话进程，包括建立、管理、终止进程间会话。
    • 🌟该层包含一种检查点机制，当断网又恢复时，通信会话会从检查点继续通信，也就是断点下载的原理。

  • 表示层

    • 主要处理不同主机中交换信息的表示方式
    • 🌟不同机器的编码和表示方法可能不同，要做到互相交换，就要由表示层采用抽象方法定义数据结构，采用标准编码形式传输，使其能相互交换信息。
    • 🌟数据压缩、加密解密也是表示层的功能。

    全书的会话层和表示层就这三个考点

  • 应用层

    • 是OSI模型的最高层，是用户与网络的接口。
    • 采用不同应用协议，满足不同类型的应用要求，因此这一层最复杂，协议也最多。

TCP/IP模型

自底向上分为网络接口层、网际层、传输层、应用层。

其中，网络接口层对应OSI中的物理层和数据链路层，应用层对应OSI中的会话层、表示层和应用层。

  • 网络接口层
    • 作用：从主机或节点接收IP分组，并发送到指定物理网络上。
    • TCP/IP没有指出网络接口层的功能和协议，仅指出主机必须使用某种协议与网络连接。
  • 网际层
    • TCP/IP模型的关键部分
    • 与OSI模型中网络层非常相似，将分组发到网络，并选择合适路由。
    • 传输单位：IP数据报
    • 仅提供无连接不可靠的服务（IPv4、IPv6）
  • 传输层
    • 与OSI中传输层类似
    • 提供两种服务：TCP、UDP

    这一点与OSI不同，OSI传输层仅提供可靠服务，TCP/IP两种都提供

    • TCP是面向连接的可靠服务，要首先建立连接，传输单位是报文段
    • UDP是无连接的不可靠服务，尽最大努力交付，传输单位是数据报
  • 应用层
    • 包含所有的高层协议（Telnet、FTP、DNS、SMTP、HTTP……）

🌟两种模型比较

  • 相同点
    • 都采用分层的体系结构，分层功能大体相似
    • 基于独立的协议栈的概念
    • 都可解决异构网络的互联
  • 🌟不同点
    • OSI精确定义了服务、协议和接口，TCP/IP没有做到
    • 层数不一样，OSI更精细
    • OSI先有模型，后有协议规范，TCP/IP正好反过来，因此TCP/IP模型不适合其它非TCP/IP网络。
    • 🌟OSI模型在网络层支持无连接和面向连接的通信，但在传输层仅有面向连接的通信
    • 🌟TCP/IP模型认为可靠性是端到端的问题，因此在网际层仅提供无连接通信，但传输层支持面向连接和无连接通信。

通信基础

基本概念

  • 数据、信号、码元
    • 数据：传送信息的实体
    • 信号：数据的电气或电磁表现，是数据在传输过程中的存在形式
      • 模拟数据（模拟信号）的取值是连续的，数字数据（数字信号）的取值是离散的。
    • 🌟在通信系统中，常用一个固定时长的信号波形表示一个k进制数，这个时长内的信号称为码元（也叫k进制码元）
      • 这个时长也叫码元宽度（信号周期）
      • 一码元可携带若干比特的信息量，体现在一个信号周期内出现k种信号，则该信号对应一个k进制数（$log_2(k)$bit）。
  • 信源、信道、信宿
    • 信源：发送数据的源头
    • 信宿：接收数据的终点
    • 信道：信号的传输介质
      • 按传输信号形式区分：模拟信道、数字信道
      • 按传输介质区分：无线信道、有线信道

      图中的噪声源是信道上的噪声及分散在通信系统其它各处噪声的集中表示。

    • 传输信号区分
      • 基带传输：在信道中直接传送基带信号
      • 宽带传输：将基带信号进行调制后，形成频分复用模拟信号，然后在信道上运输

      基带信号：由信源发出的未经过调制的原始电信号

    • 传输方式区分
      • 串行传输：逐个比特按序进行传输，适合长距离通信
      • 并行传输：若干比特通过多个通信信道同时传输，适合短距离通信（CPU与主存）
    • 交互方式区分
      • 单向通信：只有一个方向的通信，如广播等
      • 半双工通信：通信双方都可发送或接收，但任何一方都不能同时发送和接收消息（对讲机通话）
      • 全双工通信：通信双方都可同时发送或接收消息（以太网）

      单向通信只需一个信道；半双工和全双工都需要两个信道，每个方向一个信道。

  • 速率、波特、带宽
    • 速率：数据传输速率，有两种描述形式
      • 码元传输速率：又叫波特率或调制速率，单位为波特（Baud）。码元速率与进制无关。
      • 信息传输速率：又叫比特率。

      波特和比特完全不同，但波特率和比特率在数量上有关系。

      若一个码元携带n比特信息，则M Baud = Mn b/s

    • 带宽（频率带宽）表示某个信道所能传输信号的频率范围，即最高频率与最低频率之差。单位为赫兹（Hz）。

      在计网中，带宽更常见用来描述网络通信线路所能传输数据的能力，即最高数据率，此时带宽的单位为b/s

🌟信道的极限容量

任何信道都有噪音，虽然接收端能够从失真信号波形中识别出原来信号，但波特率越高，或信号传输距离越远，或传输介质质量越差，接收端波形失真越严重，直到无法读取。

  • 奈奎斯特定理：理想信道的最大波特率
    • 🌟在理想低通信道中（没有噪音、带宽有限），为了避免码间串扰，极限波特率为2W波特，其中W是信道的频率带宽（单位Hz）

      码间串扰：具体的信道所能通过的频率范围往往是有限的，信号中许多高频分量往往不能通过信道，否则在传输过程中就会衰减，导致接收端收到的信号波形失去码元之间的清晰界限，即发生码间串扰。

    • 若此时有V种不同码元，则极限数据传输速率为$2Wlog_2V$b/s

      注意单位区分

    • 结论：
      1. 在任何信道中，码元传输速率有上限，若超过这个上限，则会出现严重的码间串扰。
      2. 信道带宽越大，传输码元能力越强
      3. 奈奎斯特定理给出了波特率的限制，但没有限制码元可以携带多少比特

      若要提高数据传输速率，就要设法使每个码元携带更多比特信息量，此时需要采用多元制的调制方法。

      事实上，香农定理表明，一个码元可以携带的比特数也是有限的，只是奈奎斯特定理没有体现出来。

  • 香农定理：带宽受限，且有高斯噪声干扰的信道的极限数据传输速率
    • 若W为信道频率带宽（Hz），S为信道内所传输信号的平均功率，N为信道内高斯噪声的功率，那么信道的极限数据传输速率为$Wlog_2(1+S/N)$b/s。
      • S/N为信噪比，有两种表示方式：无单位与分贝记法（dB）。
      • 换算：若$n$为无单位信噪比，则分贝记法为$10log_{10}(n)$dB。香农定理中S/N采用无单位记法，需要学会互相换算。
    • 结论：
      1. 信道的带宽或信道中的信噪比越大，极限传输速率越高
      2. 对一定带宽与信噪比，信息传输速率的上限是固定的
      3. 只要信息传输速率低于信道极限传输速率，就能找到某种方法实现无差错的传输。
      4. 香农定理得出的是极限信息传输速率，实际信道能达到的传输速率要比它低不少。
  • 二者对比
    • 奈奎斯特定理只考虑了带宽与极限码元传输速率之间的关系，而香农定理不仅考虑了带宽，还考虑了信噪比。
    • 从另一个角度来看，一个码元可以携带的比特数是有限的。

⚠️当给定条件要求计算信道最大传输速率时，即使给出信噪比，两个公式也要同时用，并且奈奎斯特定理还要根据码元信号种数换算成极限传输速率，最后两个值比较，谁最小谁就是极限传输速率。
