数据结构的基本概念

  1. 逻辑结构

    逻辑结构是指数据对象中数据元素之间的相互关系，即从逻辑上描述数据元素之间的关系。

    注意与存储结构之间做区分

    逻辑结构分为线性结构和非线性结构，再细分为集合结构、线性结构、树形结构和图形结构。

    基本上是判断一个数据结构是属于哪种逻辑结构，所以心中有数就好。

  2. 存储结构

    存储结构则是指数据对象中数据元素之间的存储关系，即从物理上描述数据元素之间的关系。

    逻辑结构是你看到的，存储结构是计算机看到的。

    存储结构分为

    • 顺序存储结构：逻辑上相邻的元素在物理上也相邻
    • 链式存储结构：逻辑上相邻的元素在物理上不一定相邻
    • 索引存储结构：在存储结构中增加附加的索引表
    • 散列存储结构：根据关键字直接计算出存储地址

  3. 数据的运算 数据的运算是指对数据对象进行的操作，包括运算的定义和运算的实现两个方面。

    注意：运算的定义是针对逻辑结构的（如树的遍历等），运算的实现是针对存储结构的（对树的遍历来说，顺序存储和链式存储算法实现一定不同）。

  4. 注意点：

    1. 数据的逻辑结构独立于存储结构

      图的邻接表和邻接矩阵都可以表示图这种逻辑结构。

    2. 在存储数据时，不仅要存储数据元素本身，还要存储数据元素之间的逻辑关系。

时间复杂度和空间复杂度 (⭐)

每年必考，常见于选择题第一题和算法题第三小问。其中算法题主要是去研究你写的算法的时空复杂度，选择题则是考察一段代码的时空复杂度。

  1. 时空复杂度的加法规则和乘法规则

    1. 加法规则：$T(n) = O(max(f(n),g(n)))$

      两个算法串行执行时，时间复杂度取最大值

      void func1(int n){
    for (int i = 0; i < n; i++) {
        printf("Hello World\n");
    } // O(n)

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            printf("Hello World\n");
        }
    } // O(n^2)
} // func的时间复杂度为O(n^2)

    2. 乘法规则：$T(n) = O(f(n) \times g(n))$

      两个算法嵌套执行时，时间复杂度相乘
